# DESCF Submission Volunteer Workflow

## Decision

Sanity custom role is not available for this project at the moment.

Therefore, volunteers must not be invited to Sanity Studio as general Editors.

Reason:
- General Editor access may expose website content, settings, redirects, governance pages, programmes, partners, policies, newsroom, and other sensitive institutional content.
- Hiding sidebar items in Studio is only user-interface organization. It is not security.
- Real security must come from Sanity Manage roles/permissions.

## Approved Workflow

### 1. Admin exports submissions

Admin opens:

https://studio.descf.org

Then:

1. Go to **Export CSV**
2. Export **Contact Submissions**
3. Export **Event Registrations**
4. Share the CSV/Excel/Google Sheet with the assigned volunteer

### 2. Volunteer reviews submissions outside Sanity

Volunteer works only in Excel or Google Sheets.

Allowed volunteer tasks:
- Review submitted names, emails, phone numbers, message, organization, event answers
- Mark workflow status
- Add follow-up notes
- Flag suspicious, duplicate, incomplete, or irrelevant entries

Volunteer must not:
- Access Sanity Studio
- Edit website content
- Change event pages
- Change registration forms
- Change redirects/settings
- Delete submissions
- Publish public website content

### 3. Approved workflow statuses

Use only these statuses:

- New
- In Review
- Contacted
- Confirmed
- Rejected / Not Eligible
- Needs Follow-up
- Archived

### 4. Recommended spreadsheet columns

Use these columns when sharing submissions with volunteers:

- Submission ID
- Submission Type
- Submitted At
- Name
- Email
- Phone
- Organization
- Event
- Message / Answers
- Workflow Status
- Assigned To
- Internal Notes
- Last Reviewed At
- Admin Final Update Done

### 5. Admin updates Sanity

After volunteer review, admin opens Sanity Studio:

https://studio.descf.org

Then admin updates:
- Workflow Status
- Assigned To
- Internal Notes
- Last Reviewed At

Only admin should make final updates in Sanity.

## Security Rule

Until Sanity custom roles/document-level permissions are available, volunteer access must remain outside Sanity.

This protects DESCF website governance, public content, redirects, homepage curation, and institutional credibility.
